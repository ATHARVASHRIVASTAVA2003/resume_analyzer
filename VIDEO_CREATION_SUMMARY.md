## 🚀 Quick Start Guide

### Step 1: Prepare Environment (15 minutes)
```bash
# Activate virtual environment
source venv/bin/activate  # Mac/Linux
venv\Scripts\activate     # Windows

# Install dependencies
pip install -r requirements.txt

# Generate data
python -m src.generate_data

# Train model
python -m src.train_model

# Test everything
python quick_test.py
streamlit run app.py
```

### Step 2: Setup Recording (10 minutes)
1. Install screen recorder (OBS Studio recommended)
2. Increase IDE font size to 14-16pt
3. Switch to light theme
4. Open all code files in tabs
5. Clear terminal
6. Disable notifications
7. Test microphone

### Step 3: Practice (30 minutes)
1. Read through CODE_DEMO_VIDEO_SCRIPT.md
2. Practice narration without recording
3. Practice with recording (dry run)
4. Time yourself (aim for 4:30-5:00)
5. Refine sections that run long
6. Practice demo execution

### Step 4: Record (5 minutes)
1. Follow SCREEN_RECORDING_SCRIPT.md exactly
2. Speak clearly and confidently
3. Show key code sections
4. Run live demo
5. End with key takeaways

### Step 5: Review & Upload (20 minutes)
1. Watch entire video
2. Check audio quality
3. Verify code readability
4. Confirm timing (4-5 minutes)
5. Upload to YouTube or Google Drive
6. Test sharing link

## 🎤 Delivery Tips

### Voice:
- **Pace:** Moderate (not too fast)
- **Volume:** Clear and audible
- **Tone:** Confident and enthusiastic
- **Pauses:** 2 seconds between sections

### Language:
- Use "I" statements (I built, I chose, I implemented)
- Explain WHY, not just WHAT
- Highlight innovations
- Show business value

### Avoid:
- ❌ "Um", "uh", "like"
- ❌ Apologizing for code
- ❌ Reading code line by line
- ❌ Technical jargon without explanation
- ❌ Going over 5 minutes
